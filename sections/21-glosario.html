Glosario

Diccionario enciclopédico del Programa de Modernización IBS 2026

Estrategia y Gestión

Big Bang Migration
Enfoque de "todo o nada" descartado por su alto riesgo. Se prefiere la migración por oleadas (Waves).
Drivers de Esfuerzo
Factores que determinan el costo del proyecto, como la cantidad de SPs a migrar y la complejidad de los flujos multi-dominio.
Modernización Evolutiva
Estrategia de cambiar el sistema por partes mientras sigue operando, evitando interrupciones al negocio bancario.
Patrón Strangler (Estrangulador)
Metodología donde las nuevas funcionalidades en .NET van "envolviendo" al legado hasta que este desaparece.
Wave (Oleada)
Conjunto de releases que agrupan módulos funcionales (ej. Wave 1: Préstamos, Wave 2: Depósitos) según el plan 2026.
WBS (Work Breakdown Structure)
Descomposición del proyecto en tareas manejables: Gestión, Arquitectura, Plataforma, Datos, Migración y Testing.

Arquitectura (Monolito Modular)

ADR (Architecture Decision Record)
Registro formal de una decisión técnica para evitar "amnesia colectiva" y re-debates cíclicos.
Anticorruption Layer (ACL)
Capa que traduce datos del sistema viejo al nuevo, impidiendo que el diseño sucio del legado afecte al código limpio.
Bounded Context
Límite donde un modelo de datos es consistente. Un módulo de "Clientes" no es igual al de "Riesgos".
Modularidad Fuerte
Capacidad de cambiar un módulo internamente sin afectar a los demás, lograda mediante interfaces claras.
Quiescence
Estado de reposo necesario para migrar datos de un módulo sin que haya transacciones activas.
Trade-off
Decisión donde se gana algo (ej. escalabilidad) perdiendo otra cosa (ej. simplicidad). No hay soluciones perfectas.

Datos e Integración

ACID
Propiedad no negociable para el Core Bancario: Atomicidad, Consistencia, Aislamiento y Durabilidad.
  • Atomicidad: La transacción es indivisible; se completa totalmente o se revierte íntegramente ante cualquier error.
  • Consistencia: Asegura que la transacción lleve la base de datos de un estado válido a otro, cumpliendo todas las reglas de integridad bancaria.
  • Aislamiento: Las transacciones simultáneas se ejecutan sin interferir entre sí, evitando errores de saldo o lecturas sucias.
  • Durabilidad: Una vez confirmada, la transacción persiste de forma permanente, incluso ante fallos críticos de hardware o energía.
Compensación (SAGA)
Acción de reversa que se ejecuta si un paso intermedio de una transacción distribuida falla.
Consistencia Eventual
Modelo donde los datos se sincronizan tras un breve lapso de tiempo, aceptable en flujos no contables.
Idempotencia
Garantía de que procesar el mismo mensaje dos veces no duplique el efecto (ej. no cobrar dos veces el mismo interés).
Outbox Pattern
Técnica para asegurar que los eventos se guarden en la DB antes de enviarse a una cola, evitando pérdida de datos.
SAGA Pattern
Gestión de transacciones largas que abarcan múltiples módulos sin usar bloqueos pesados de base de datos.
Schema Firewall
Reglas y contratos para evitar que aplicaciones externas o procesos legados corrompan el nuevo esquema de datos.

Legado y Tecnología

Coexistencia
Periodo crítico donde PB y .NET operan sobre la misma base de datos Sybase simultáneamente.
Deuda Técnica
Costo acumulado por años de parches en PowerBuilder que dificultan el mantenimiento actual.
Lógica In-DB
Código de negocio escrito directamente en Stored Procedures (SPs), el cual se busca extraer hacia servicios C#.
PowerBuilder (PB)
Lenguaje del sistema IBS actual. Su obsolescencia es el driver principal del programa de modernización.
Sybase ASE
Motor de base de datos actual. La estrategia busca optimizar el acceso desde la nueva arquitectura .NET.

Métricas de Éxito (KPIs)

Defect Leakage
Porcentaje de fallos que llegan a manos del usuario. Meta: < 5% por cada release.
MTTR (Mean Time To Repair)
Tiempo promedio de resolución de incidentes críticos (P1/P2) detectados durante el piloto.
P95 Latency
El tiempo de respuesta que no es superado por el 95% de las transacciones. Mide la velocidad real percibida.
Release Readiness Score
Criterios de salida que incluyen cobertura de tests (>80%), seguridad y performance aprobados.
Tasa de Compensación
Porcentaje de SAGAs que fallan y requieren reversa. Un número alto indica inestabilidad en el diseño.
