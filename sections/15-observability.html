Observabilidad

Ver qué pasa adentro del sistema sin necesidad de reiniciarlo

Observabilidad = capacidad de entender el estado interno de un sistema a partir de sus outputs externos. Los tres pilares son:

📝

Logs Estructurados

Logs en formato JSON (no texto libre). Permiten búsqueda, filtrado y agregación en herramientas como Elasticsearch o Splunk.

🔍

Trazabilidad (Tracing)

Seguir una transacción de punta a punta a través de todos los módulos. Requiere propagar un Correlation ID.

📊

Métricas

Datos numéricos en el tiempo: latencia p95/p99, tasa de errores, throughput, uso de memoria. Expuestos via Prometheus/Grafana.

Correlation ID — La clave de la trazabilidad

Un identificador único que se genera en la entrada del sistema y se propaga a todos los módulos, logs y eventos de esa transacción.

// Middleware que asegura que toda request tenga correlation ID
public class CorrelationIdMiddleware
{
    public async Task InvokeAsync(HttpContext context)
    {
        // Tomar del header o generar uno nuevo
        var correlationId = context.Request.Headers["X-Correlation-ID"]
            .FirstOrDefault() ?? Guid.NewGuid().ToString();

        // Disponible en toda la request
        context.Items["CorrelationId"] = correlationId;
        context.Response.Headers["X-Correlation-ID"] = correlationId;

        // Propagar en todos los logs de esta request
        using (LogContext.PushProperty("CorrelationId", correlationId))
        {
            await _next(context);
        }
    }
}

// Log estructurado — todos llevan CorrelationId
_logger.Information(
    "Transferencia {TransferenciaId} ejecutada por {ClienteId} por {Monto}",
    transferencia.Id, cmd.ClienteId, cmd.Monto);
