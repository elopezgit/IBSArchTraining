Observabilidad: El sistema bajo rayos X

Monitoreo proactivo para sistemas de alta disponibilidad.

La observabilidad no es solo "hacer logs". Es la capacidad de responder cualquier pregunta sobre el comportamiento del sistema sin tener que desplegar código nuevo. En el IBS 2026, pasamos del "creo que falló la base de datos" al "el 95% de las peticiones fallan en el módulo de Cuentas por un timeout en la tabla de Reservas".

Los Tres Pilares (The Golden Signals)

📝

Logs Estructurados

Olvidémonos de Console.WriteLine. Usamos Serilog para generar JSONs con propiedades.
Poder: Puedes filtrar por ClienteId en segundos entre millones de líneas.

🔍

Trazas (Distributed Tracing)

Visualiza el viaje de una petición entre módulos.
Poder: Ves exactamente cuántos milisegundos pasó la petición en Pagos, cuántos en Cuentas y cuánto tardó la DB de Sybase.

📊

Métricas

Agregados numéricos.
Poder: Tableros en tiempo real con la cantidad de transacciones por segundo (TPS) y la tasa de error actual.

La Columna Vertebral: Correlation ID

En una arquitectura modular, una operación de "Transferencia" genera logs en 4 servicios distintos. Sin un Correlation ID (o Trace ID), es imposible unir las piezas.

// Implementación automática en .NET 8 usando OpenTelemetry
builder.Services.AddOpenTelemetry()
    .WithTracing(tracing => tracing
        .AddSource("IBS.Core")
        .AddAspNetCoreInstrumentation()
        .AddHttpClientInstrumentation() // Propaga el ID a otros módulos
        .AddSqlClientInstrumentation()); // Rastrea tiempos de SQL

Métricas Críticas (KPIs de Ingeniería)

Para nivelar el contenido, el equipo debe conocer los "Four Golden Signals" de Google SRE, adaptados a IBS:

Señal Qué mide en IBS Umbral de Alerta
Latencia (p99) Tiempo que tarda el 99% de las transferencias. > 2.0 segundos
Tráfico (Throughput) Cantidad de peticiones por segundo (TPS). Caída repentina de -30%
Errores Tasa de HTTP 500 o excepciones de negocio. > 1% de las peticiones
Saturación Uso de CPU/RAM o conexiones al Pool de Sybase. > 80% de capacidad

Ejemplo: Del Log al Diagnóstico

Un log estructurado profesional se ve así en nuestra plataforma de observabilidad:

{
  "Timestamp": "2026-04-24 16:00:00",
  "Level": "Error",
  "Message": "Falla en reserva de saldo para transferencia {TxId}",
  "Properties": {
    "CorrelationId": "b55a12...",  // <-- El "hilo" que une todo
    "Module": "Accounts",
    "TxId": "TX-9988",
    "ExecutionTimeMs": 1250,
    "ErrorCode": "INSUFFICIENT_FUNDS",
    "Environment": "Production"
  }
}
💡
¿Por qué esto es vital para IBS? Cuando migramos módulos de Sybase a .NET, la observabilidad nos permite comparar: "¿Es el nuevo código más rápido que el anterior?". Si no medimos, solo estamos adivinando.

Regla de Oro de Logging:

Nunca loguear datos sensibles (PII). Está prohibido por regulaciones bancarias guardar números de tarjeta, CVV o saldos en texto claro dentro de los logs. Usa máscaras: 4532-XXXX-XXXX-1234.
