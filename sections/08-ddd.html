DDD Práctico

Domain-Driven Design — el lenguaje para modelar el dominio bancario

DDD (Eric Evans, 2003) es un enfoque de diseño que pone el modelo de dominio en el centro. El código habla el mismo idioma que el negocio.

Bloques de Construcción

🏛️ Entity (Entidad)

Objeto con identidad única que persiste en el tiempo. Ej: Cuenta, Cliente, Transferencia. Dos cuentas con distinto ID son objetos distintos aunque tengan el mismo saldo.

💎 Value Object

Objeto definido por sus atributos, sin identidad. Son inmutables. Ej: Money, IBAN, Periodo. Dos Money(100, ARS) son iguales e intercambiables.

🫙 Aggregate

Grupo de entidades que forman una unidad de consistencia. Solo se accede desde la raíz (Aggregate Root). Ej: CuentaCorriente con sus movimientos.

⚙️ Domain Service

Operaciones de dominio que no pertenecen naturalmente a una entidad. Ej: CalculadoraIntereses, ValidadorLimitePagos.

📡 Domain Event

Algo que ocurrió en el dominio y otros módulos pueden necesitar saber. Ej: CuentaAbierta, TransferenciaEjecutada, PrestamoAprobado.

🗄️ Repository

Interfaz para persistir y recuperar Aggregates. El Dominio define qué necesita; la Infraestructura implementa cómo.

Ejemplo completo: Módulo Cuentas

// VALUE OBJECT — inmutable, sin identidad
public record Money(decimal Amount, Currency Currency)
{
    public Money Add(Money other)
    {
        if (Currency != other.Currency) throw new InvalidOperationException();
        return new Money(Amount + other.Amount, Currency);
    }
    public static bool operator >=(Money a, Money b) => a.Amount >= b.Amount;
}

// AGGREGATE ROOT — unidad de consistencia
public class CuentaCorriente : AggregateRoot<CuentaId>
{
    public ClienteId Titular { get; private set; }
    public Money Saldo { get; private set; }
    private List<Movimiento> _movimientos = new();

    // Lógica de negocio dentro del aggregate
    public void Debitar(Money monto, string concepto)
    {
        if (Saldo < monto)
            throw new SaldoInsuficienteException(Id, monto);
        Saldo = Saldo - monto;
        _movimientos.Add(Movimiento.Debito(monto, concepto));
        AddDomainEvent(new SaldoActualizadoEvent(Id, Saldo));
    }
}

// DOMAIN SERVICE — opera sobre múltiples aggregates
public class ServicioTransferencia
{
    public Result Transferir(CuentaCorriente origen, 
                              CuentaCorriente destino, 
                              Money monto)
    {
        origen.Debitar(monto, "Transferencia saliente");
        destino.Acreditar(monto, "Transferencia entrante");
        return Result.Success();
    }
}
