Resiliencia y Estabilidad

Diseñando para el fallo: El arte de recuperarse con elegancia.

En el IBS moderno, asumimos que la red es traicionera y que los servicios externos (como el Buró de Crédito o el validador de identidad) fallarán. La resiliencia es la capacidad del sistema de absorber impactos y seguir operando.

Patrones de Diseño para la Robustez

🔄

Retry (Reintento)

Filosofía: "Si fallas, intenta de nuevo, pero con calma".
Usamos Exponential Backoff: esperamos 1s, luego 2s, luego 4s... para no "rematar" a un servidor que ya está sufriendo.

⚡

Circuit Breaker

Filosofía: "Si el vecino no responde, deja de tocarle el timbre".
Si un servicio falla N veces, el circuito se abre y las llamadas fallan instantáneamente, ahorrando recursos y permitiendo que el servicio se cure.

🛡️

Bulkhead (Mamparos)

Filosofía: "Si se inunda un compartimento del barco, que no se hunda todo".
Aislamos los recursos (hilos/memoria) por módulo. Si Pagos está saturado, Consultas debe seguir funcionando.

Implementación con Polly en .NET 8

Polly es nuestra herramienta principal. Definimos "Estrategias de Resiliencia" que envuelven nuestras llamadas a otros módulos o servicios externos.

// Configuración de una política combinada (Resilience Pipeline)
var pipeline = new ResiliencePipelineBuilder<HttpResponseMessage>()
    .AddRetry(new RetryStrategyOptions<HttpResponseMessage> {
        MaxRetryAttempts = 3,
        BackoffType = DelayBackoffType.Exponential,
        UseJitter = true // Añade aleatoriedad para evitar el "thundering herd"
    })
    .AddCircuitBreaker(new CircuitBreakerStrategyOptions<HttpResponseMessage> {
        FailureRatio = 0.5, // Abre si el 50% de llamadas fallan
        SamplingDuration = TimeSpan.FromSeconds(30),
        MinimumThroughput = 10
    })
    .Build();

// Ejecución segura
await pipeline.ExecuteAsync(async ct => await _httpClient.GetAsync(url, ct));

Estrategias de Contingencia (Fallback)

¿Qué hacemos cuando el Circuit Breaker está abierto? Aquí es donde entra el diseño de negocio:

Servicio Fallido Estrategia de Fallback (Plan B)
Validador de Identidad Derivar a validación manual por un operador humano.
Motor de Scoring Usar el último score guardado en caché (si tiene < 24hs).
Servicio de Notificaciones Encolar el mensaje en la base de datos para envío posterior.
🚨
Cuidado con el Reintento Infinito: Reintentar operaciones que no son Idempotentes (como un POST de transferencia) sin una llave de idempotencia puede causar duplicidad de cargos. El Retry y la Idempotencia deben ir siempre de la mano.
💡
Observabilidad y Resiliencia: Cada vez que un Circuit Breaker cambia de estado (de Cerrado a Abierto), debemos disparar una alerta crítica. Es el sistema diciéndonos: "No puedo más, estoy bloqueando tráfico para sobrevivir".

Resumen de Nivelación:

Un sistema resiliente no es el que nunca falla, sino el que sabe fallar rápido y recuperarse solo. En IBS, aplicamos Polly en cada cliente HTTP y en cada llamada entre módulos para asegurar que un error local no se convierta en una caída nacional.
