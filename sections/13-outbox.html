Patrón Outbox

Garantizar la publicación confiable de eventos junto con cambios de datos

El Problema

Guardar en la BD y publicar un evento son dos operaciones separadas. Entre ellas puede fallar el proceso.

Problema sin Outbox — Riesgo de pérdida de eventos
                        ❌ PROCESO FALLA AQUÍ ❌
                                │
Módulo Pagos ──► Guardar en BD ─┤─► Publicar evento
                   (exitoso)    │   (nunca ocurre)
                                │
Resultado: BD actualizada, Contabilidad no se enteró

La Solución: Transactional Outbox

Persistir el evento en la misma transacción que los datos, en una tabla outbox. Un proceso separado (Relay) publica los eventos pendientes.

Patrón Outbox — Garantía at-least-once delivery
 STEP 1: Una sola transacción atómica
 ┌────────────────────────────────────────────┐
 │  BEGIN TRANSACTION                        │
 │    UPDATE cuentas SET saldo = saldo - 100  │
 │    INSERT INTO outbox                      │
 │      (type='TransferenciaEjecutada',       │
 │       payload='{...}', published=false)    │
 │  COMMIT TRANSACTION                        │
 └────────────────────────────────────────────┘

 STEP 2: Proceso Relay (background)
 ┌────────────────────────────────────────────┐
 │  SELECT * FROM outbox                      │
 │  WHERE published = false                   │
 │  ORDER BY created_at                       │
 │                                            │
 │  FOR EACH evento:                          │
 │    → Publicar al Event Bus                 │
 │    → UPDATE outbox SET published=true      │
 └────────────────────────────────────────────┘

Implementación en .NET

// 1. Tabla outbox (misma BD que el módulo)
public class OutboxMessage
{
    public Guid Id { get; set; } = Guid.NewGuid();
    public string Type { get; set; }       // "TransferenciaEjecutada"
    public string Payload { get; set; }    // JSON serializado
    public bool Published { get; set; }
    public DateTime CreatedAt { get; set; }
}

// 2. En el Application Service — guardar dato + outbox en una transacción
public class EjecutarTransferenciaHandler
{
    public async Task<Result> Handle(TransferirCmd cmd)
    {
        await using var tx = await _db.BeginTransactionAsync();
        try
        {
            // Operación de negocio
            var transferencia = await _service.Ejecutar(cmd);

            // Guardar evento en outbox — MISMA transacción
            _db.OutboxMessages.Add(new OutboxMessage
            {
                Type = "TransferenciaEjecutada",
                Payload = JsonSerializer.Serialize(transferencia),
            });

            await _db.SaveChangesAsync();
            await tx.CommitAsync();
            return Result.Success();
        }
        catch
        {
            await tx.RollbackAsync();
            throw;
        }
    }
}

// 3. Relay — background service que publica lo pendiente
public class OutboxRelayService : BackgroundService
{
    protected override async Task ExecuteAsync(CancellationToken ct)
    {
        while (!ct.IsCancellationRequested())
        {
            var pending = await _db.OutboxMessages
                .Where(m => !m.Published).ToListAsync();
            foreach (var msg in pending)
            {
                await _eventBus.Publish(msg.Type, msg.Payload);
                msg.Published = true;
            }
            await _db.SaveChangesAsync();
            await Task.Delay(1000, ct);
        }
    }
}
