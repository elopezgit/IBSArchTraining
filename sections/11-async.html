Comunicación Asíncrona

El módulo publicador no espera respuesta. Publica un mensaje/evento y continúa. Los suscriptores reaccionan cuando pueden.

Event-Driven — Transferencia ejecutada notifica Contabilidad

Tipos de Comunicación Asíncrona

📨 Cola (Queue)

Point-to-point. Un productor, un consumidor. Cada mensaje es procesado por exactamente un receptor. Ideal para comandos: "Procesar esta transferencia".

📡 Topic / Event

Pub/Sub. Un productor, múltiples consumidores. Todos los interesados reciben la copia. Ideal para eventos: "TransferenciaEjecutada" → notifica Contabilidad, Notificaciones, Reportes.

Implementación en .NET (MediatR + eventos de dominio)

// 1. Definir el Domain Event
public record TransferenciaEjecutadaEvent(
    TransferenciaId Id,
    Money Monto,
    CuentaId Origen,
    CuentaId Destino,
    DateTime FechaEjecucion
) : IDomainEvent;

// 2. Publicar desde el Aggregate Root (automático al guardar)
public class CuentaCorriente : AggregateRoot
{
    public void Debitar(Money monto)
    {
        // ... lógica
        AddDomainEvent(new TransferenciaEjecutadaEvent(...));
    }
}

// 3. Handler en Contabilidad — reacciona al evento
public class RegistrarAsientosHandler
    : INotificationHandler<TransferenciaEjecutadaEvent>
{
    public async Task Handle(TransferenciaEjecutadaEvent evt,
                             CancellationToken ct)
    {
        // Registrar débito y crédito en GL
        await _glService.RegistrarAsientos(
            AsientoDebito(evt.Origen, evt.Monto),
            AsientoCredito(evt.Destino, evt.Monto)
        );
    }
}
